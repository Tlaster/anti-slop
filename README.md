# Anti Slop

Anti Slop is a JavaScript GitHub Action that scores pull request authors for
high-volume, template-like contribution patterns. It is designed for manual
review gates: surface suspicious PRs, expose the exact signals, and optionally
fail the check when the score crosses a threshold.

## Usage

```yaml
name: Anti Slop

on:
  pull_request:
    types: [opened, reopened, synchronize]

permissions:
  contents: read
  issues: write
  pull-requests: write

jobs:
  anti-slop:
    runs-on: ubuntu-latest
    steps:
      - uses: Tlaster/anti-slop@main
        with:
          github-token: ${{ github.token }}
          fail-on-threshold: 'true'
          close-pr-on-threshold: 'false'
          close-linked-author-issues: 'true'
          fail-threshold: '100'
```

Set `close-pr-on-threshold: 'true'` to close PRs that meet or exceed
`fail-threshold`. When closing a PR, the action can also close issues referenced
by GitHub closing keywords such as `Closes #123`, but only when the issue was
opened by the same author as the PR. Closing PRs requires
`pull-requests: write`; closing linked issues requires `issues: write`. For
public repositories that need to close PRs from forks, use `pull_request_target`
carefully and do not check out or execute untrusted PR code in the same job.

## Scoring

The action adds configurable weights when a PR author trips these rules:

- `one_day_pr_volume`: PR count in the short rolling window exceeds
  `one-day-pr-threshold`
- `seven_day_pr_volume`: PR count in the broader rolling window exceeds
  `seven-day-pr-threshold`
- `cross_repo_volume`: sampled recent PRs span more repositories than
  `cross-repo-threshold`
- `template_marker_ratio`: enough sampled PR bodies contain configured
  `template-markers`
- `untrusted_author_association`: PR `author_association` is not in
  `trusted-author-associations`
- `no_prior_merged_prs`: author has fewer merged PRs in this repository than
  `prior-merged-pr-threshold`

The default weights are tuned so a user with very high PR volume across many
repositories, templated bodies, no association, and no prior merged history will
cross the default high-risk threshold.

## Inputs

| Input                             | Default                                                |
| --------------------------------- | ------------------------------------------------------ |
| `github-token`                    | empty, falls back to `GITHUB_TOKEN`                    |
| `fail-on-threshold`               | `false`                                                |
| `close-pr-on-threshold`           | `false`                                                |
| `close-linked-author-issues`      | `true`                                                 |
| `fail-threshold`                  | `100`                                                  |
| `one-day-window-hours`            | `24`                                                   |
| `one-day-pr-threshold`            | `20`                                                   |
| `seven-day-window-days`           | `7`                                                    |
| `seven-day-pr-threshold`          | `100`                                                  |
| `sample-size`                     | `100`                                                  |
| `cross-repo-threshold`            | `30`                                                   |
| `template-markers`                | `Made with [Cursor]`, `Generated with`, `Generated by` |
| `template-marker-ratio-threshold` | `0.3`                                                  |
| `trusted-author-associations`     | `OWNER,MEMBER,COLLABORATOR,CONTRIBUTOR`                |
| `prior-merged-pr-threshold`       | `1`                                                    |
| `weight-one-day-prs`              | `40`                                                   |
| `weight-seven-day-prs`            | `40`                                                   |
| `weight-cross-repo`               | `30`                                                   |
| `weight-template-markers`         | `15`                                                   |
| `weight-untrusted-association`    | `10`                                                   |
| `weight-no-prior-merged`          | `20`                                                   |

## Outputs

- `score`
- `risk-level`
- `triggered-rules`
- `details`
- `one-day-pr-count`
- `seven-day-pr-count`
- `unique-repo-count`
- `template-marker-count`
- `prior-merged-pr-count`
- `author-association`
- `pr-closed`
- `linked-issues-closed`
- `linked-issue-refs`

## Local Development

```bash
npm install
npm test
npm run package
```

`dist/index.js` is committed because JavaScript actions run from the packaged
entrypoint.
